# 3 Tipos de cardinalidad

1-1
Un solo elemento de una lista se puede conectar con un solo elemento de otra lista

Una persona tiene **UN** DNI
(Cuándo esto ocurre, esos dos datos deberían estar en una sola tabla)
---

1-n (Uno a muchos)

Una persona con muchos hobbies
Un cliente con muchos pedidos
---

n-n (de muchos a muchos)

A muchos elementos de una lista le corresponden muchos elementos de ota lista.
La solución suele ser una tabla intermediaria

Una asignatura puede tener multiples alumnos
U alumno puede tener multiples asignaturas
---
