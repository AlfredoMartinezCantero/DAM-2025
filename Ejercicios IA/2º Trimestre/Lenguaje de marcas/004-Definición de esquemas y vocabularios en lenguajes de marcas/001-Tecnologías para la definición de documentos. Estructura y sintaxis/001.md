# Introducción:
En este ejercicio práctico he trabajado con la definición de documentos XML, este a diferencia de HTML que tiene etiquetas fijas, XML permite etiquetas personalizables para estructurar información, para resolver los ejercicios, he comenzado asegurando la existencia de una etiqueta raíz única, que en este caso es `<persona>`, dentro la raíz he definido subetiquetas para organizar la información básica como `<nombre> y <apellidos>`, y he añadido un nuevo campo `<email>` siguiendo la estructura jerárquica padre-hijo, por último para el manejo de datos, como los números de teléfono no he repetido la etiqueta `<telefono>` suelta, sino que he utilizado una estructura de lista, es decir, he creado un elemento contenedor `<telefonos>` y, dentro de él, he anidado múltiples elementos `<telefono>`, lo que permite agrupar lógicamente la información repetida sin romper la estructura del documento.

---
```
<?xml version="1.0" encoding="UTF-8"?>
<persona>
  <nombre>Jose Vicente</nombre>
  <apellidos>Carratala Sanchis</apellidos>
  <email>ejemplo@example.com</email>

  <telefonos>
    <telefono>12345567</telefono>
    <telefono>6534646</telefono>
    <telefono>98765432</telefono>
    <telefono>666777888</telefono>
  </telefonos>
</persona>
``` 
---
# Conclusión
Con estos ejercicios he aprendido que XML es un estándar diseñado específicamente para guardar y transportar datos planos de forma estructurada la jerarquía estricta de etiquetas permite modelar objetos complejos, como una persona con múltiples teléfonos, de una forma que sea legible tanto para humanos como para máquinas
