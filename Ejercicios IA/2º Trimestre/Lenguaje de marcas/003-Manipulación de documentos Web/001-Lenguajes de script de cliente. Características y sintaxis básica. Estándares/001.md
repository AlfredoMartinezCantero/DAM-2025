# Introducción:
En este ejercicio he integrado JavaScript dentro de un documento HTML vinculando también un css externo, he empezado por instanciar la clase `Date` para obtener la fecha y hora actuales verificando su funcionamiento mediante `console.log` en la misma consola del navegador.

Después he encapsulado la lógica en una función llamda `diHola`, en lugar de hacer que la función imprima directamente en pantalla he utilizado la sentencia `return` para devolver una cadena de texto, lo que permite que la función sea reutilzable.

---
```
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Ejercicio Funciones y Fechas</title>
    <link rel="stylesheet" href="estilo.css">
</head>
<body>

    <div class="contenedor">
        <h1>Ejercicio:</h1>
        <p id="mensaje-salida">Esperando ejecución...</p>
    </div>

    <script>
        const fechaActual = new Date();
        console.log("La fecha actual es:", fechaActual);

        function diHola(nombre, edad) {
            return `Hola, ${nombre} tienes ${edad} años, ¿cómo estás?`;
        }

        const nombreUsuario = "Jose Vicente";
        const edadUsuario = 47;

        const mensajeGenerado = diHola(nombreUsuario, edadUsuario);

        const elementoSalida = document.getElementById("mensaje-salida");
        elementoSalida.textContent = mensajeGenerado;
    </script>
</body>
</html>
```
---
# Conclusión:
Con este código se consolida la diferencia entre mostrar un dato en `console.log` y devolver el dato con `return`, gracias a estas funciones consigo que el código sea modular, es decir, puedo usar `diHola` para mostrar el mensaje en una alerta, en la consola o en el HTML sin tener que modificar la lógica interna de la función.

Un error común es poner el `<script>` en el `<head>` sin usar atributos por que el script se ejecutaría antes de que el navegador leyera el `<body>`, devolviendo `null` rompiendo asi la ejecución.
