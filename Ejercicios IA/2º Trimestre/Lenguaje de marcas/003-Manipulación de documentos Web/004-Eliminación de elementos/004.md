# Introducción:
En este ejercicio he desarrollado un pequeño videojuego interactivo como "Aim Labs", pero en una fase alfa por asi decirlo, para comprender el ciclo de vida de los elementos en el DOM, he comenzado utilizando `document.createElement('div')` para generar dinámicamente nuevos elementos HTML sin necesidad de escribirlos previamente en el código base.

Para lograr que estos elementos aparezcan en lugares impredecibles, he aplicado la función `Math.random()` multiplicada por el ancho y alto de la ventana, `window.innerWidth y window.innerHeight`, asignando estos valores a las propiedades CSS top y left del elemento.

Por último, he implementado la lógica de eliminación, en lugar de borrar elementos manualmente, he asignado un `onclick` a cada nuevo elemento creado, esto lo que hace es que al detectar el clic, se ejecuta el método `this.remove()`, lo que permite que el propio objeto se elimine a sí mismo del árbol del documento.

---
```
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Práctica: Crear y Eliminar</title>
    <style>
        body {
            background-color: #222;
            color: white;
            font-family: sans-serif;
            overflow: hidden; /* Evita barras de scroll cuando los elementos salen al borde */
            margin: 0;
            height: 100vh;
            cursor: crosshair;
        }

        /* Estilo base para los objetivos */
        .objetivo {
            width: 50px;
            height: 50px;
            background-color: crimson;
            border-radius: 50%; /* Lo hace redondo */
            position: absolute; /* mover con top/left */
            border: 2px solid white;
            box-shadow: 0 0 10px red;
            transition: transform 0.1s;
        }

        .objetivo:active {
            transform: scale(0.9);
        }

        #marcador {
            position: fixed;
            top: 20px;
            left: 20px;
            font-size: 24px;
            z-index: 1000;
            pointer-events: none; /* El clic traspasa el texto */
        }
    </style>
</head>
<body>

    <div id="marcador">Puntuación: <span id="puntos">0</span></div>

    <script>
        let puntuacion = 0;
        const displayPuntos = document.getElementById('puntos');

        // Función encargada de crear un nuevo elemento
        function crearObjetivo() {
            // Crear el nodo elemento
            const nuevoElemento = document.createElement('div');
            
            // Asignar clase para estilos
            nuevoElemento.className = 'objetivo';

            // Posicionamiento Aleatorio
            // Restamos 50px para que no aparezcan fuera de la pantalla
            const x = Math.random() * (window.innerWidth - 60);
            const y = Math.random() * (window.innerHeight - 60);

            nuevoElemento.style.left = x + 'px';
            nuevoElemento.style.top = y + 'px';

            // Lógica de Eliminación
            nuevoElemento.onclick = function() {
                // Elemento que ha recibido el clic
                this.remove(); 
                
                // Actualizamos lógica del juego
                puntuacion++;
                displayPuntos.textContent = puntuacion;
            };

            // Hacer visible el DOM
            document.body.appendChild(nuevoElemento);
        }

        // Crear un objetivo cada segundo
        setInterval(crearObjetivo, 1000);
    </script>
</body>
</html>
```

# Conclusión:
Con este código demuestro el uso de la manipulación del DOM en tiempo real, comprendiendo que una página web no es estática, sino que puede cambiar añadiendo y quitando nodos según la interacción del usuario, también he aprendido que el método `remove()` es la forma más limpia y directa de eliminar elementos específicos sin afectar a sus padres o hermanos, y que combinar JavaScript con CSS `(position: absolute)` es esencial para crear interfaces dinámicas como videojuegos.

Un error común que he identificado es olvidar definir `position: absolute`, si tuviera `static` las propiedades de `top` y `left` de JS haría que todos los elementos apareciesesn apilados uno debajo de otro.
