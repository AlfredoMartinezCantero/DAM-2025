# Introducción:
Este ejercicio he aprendido a conectar la lógica de programación con la interfaz visual mediante la manipulación del `Document Object Model`, el objetivo principal es entender cómo JavaScript puede "leer" y "modificar" el HTML en tiempo real.

He utilizado el método `document.querySelector()`, que me permite "atrapar" cualquier elemento de la página usando etiquetas o IDs.

Por último, en lugar de escribir cada título del blog a mano en el HTML, he declarado un array de strings en JavaScript y he utilizado un bucle `for`, esto hace que en cada vuelta del bucle, accedo al índice correspondiente del array y concateno el código HTML necesario dentro del contenedor `<main>`.

---
```
<!DOCTYPE html>
<html lang="es">
    <head>
        <title>Javascript</title>
        <meta charset="utf-8">
    </head>
    <body>
        <p>Soy un párrafo</p>
        <p>Mi nuevo párrafo</p>
        <script>
            const elemento = document.querySelector("p");
            document.write(elemento);
            document.write(elemento.textContent);
        </script>
    </body>
</html>
```

                "Tercer artículo",

            ];
            const contenedor = document.querySelector("main");
            for(let i = 0;i<articulos.length;i++){              
                contenedor.innerHTML += "<h3>"+articulos[i]+"</h3>";
            }
        </script>
    </body>
</html>

# Conclusión:
Este ejercicio nos enseña la capacidad de alterar la página web basándome en datos, comprobando que no necesito escribir cientos de líneas de HTML si tengo los datos estructurados en un array y permitiendo añadir más artículos simplemente modificando la lista en JavaScript, sin tocar la estructura visual.

Un error común es a la hora de usar `document.write()`, ya que es una instrucción antigua que, si se usa mal, puede borrar todo el contenido de la página existente, por lo que he priorizado el uso de `querySelector`.
