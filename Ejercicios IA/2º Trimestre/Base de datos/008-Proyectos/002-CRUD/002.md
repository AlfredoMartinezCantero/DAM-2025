# Conclusión:
Con este ejercicio he aprendido a separar los datos de la lógica de consulta, al usar marcadores de posición `nombre`, delego en el driver de la base de datos la tarea de "escapar" los caracteres peligrosos, asegurando la integridad del sistema, también he comprendido la importancia de usar bloques `try-catch` para gestionar errores de conexión sin que la página web se rompa abruptamente mostrando información sensible al usuario.

Un error común en PHP es confiar en los datos que envía el usuario, es decir, en mi primer borrador utilicé la sintaxis `$$_POST` por error (lo que intenta crear variables dinámicas incontroladas) y concatené las variables directamente en el SQL `(id = ".$id.")`. Si un usuario malintencionado hubiera enviado 1 OR 1=1 en el campo ID o código SQL en el nombre, podría haber alterado todos los registros de la tabla o borrado la base de datos completa.
