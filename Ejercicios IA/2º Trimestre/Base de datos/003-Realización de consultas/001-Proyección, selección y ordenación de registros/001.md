# Introducción:
En este ejercicio se trabaja la creación y y manipulación de estructuras de datos relacionales, comenzando por la definición de una base de datos con `CREATE DATABASE` y su uso. También se el diseño de una tabla mediante `CREATE TABLE` especificando columnas de texto `VARCHAR` y numéricas `INT` para almacenar la información de los clientes.

Luego me he enfocado en el lenguaje de consulta de datos, tras poblar la tabla con `INSERT INTO`, he experimentado con la proyección de columnas específicas para optimizar la salida y he utilizado la palabra clave `AS` para generar alias, lo que me permite presentar los encabezados de forma más amigable al usuario final, por último, he usado `ORDER BY`, aplicando criterios de ordenación ascendente con `ASC` y descendente con `DESC`.

---
```
-- sudo mysql -u root -p

-- Creamos una base de datos

CREATE DATABASE clientes_ejercicio;

USE clientes_ejercicio;

CREATE TABLE clientes(
  nombre VARCHAR(255),
  apellidos VARCHAR(255),
  edad INT
);

INSERT INTO clientes VALUES('Juan', 'Pérez', 30), ('María', 'López', 25), ('Carlos', 'González', 40);

-- Ordenamos descendentemente por edad

SELECT 
  nombre AS 'Nombre del cliente',
  apellidos AS 'Apellidos del cliente',
  edad AS 'Edad del cliente'
FROM 
  clientes
ORDER BY
  edad DESC;

-- Hago una selección nombrando las columnas

SELECT 
  nombre,
  apellidos,
  edad 
FROM 
  clientes;

-- Ordenación ascendente explícito por apellidos

SELECT 
  nombre AS 'Nombre del cliente',
  apellidos AS 'Apellidos del cliente',
  edad AS 'Edad del cliente'
FROM 
  clientes
ORDER BY
  apellidos ASC;

-- Selecciono solo algunas columnas:

SELECT 
  nombre,
  apellidos
FROM 
  clientes;

-- La proyección se refiere a seleccionar un subconjunto de columnas de una tabla. 

SELECT 
  nombre AS 'Nombre del cliente',
  apellidos AS 'Apellidos del cliente'
FROM 
  clientes;

-- Hay un orden por defecto, si no se especifica el orden, SQL ordenará los resultados en orden ascendente por la columna seleccionada.

SELECT 
  nombre AS 'Nombre del cliente',
  apellidos AS 'Apellidos del cliente',
  edad AS 'Edad del cliente'
FROM 
  clientes
ORDER BY
  apellidos;

-- También se puede seguir un orden por dos columnas, se pueden ordenar los resultados por más de una columna, especificando el orden para cada una.

SELECT 
  nombre AS 'Nombre del cliente',
  apellidos AS 'Apellidos del cliente',
  edad AS 'Edad del cliente'
FROM 
  clientes
ORDER BY
  edad DESC,apellidos ASC;
```

# Conclusión:
Con este ejercicio comprendemos sobre cómo el motor de base de datos recupera y presenta la información,  almacenando los datos es solo el primer paso, el verdadero valor reside en saber extraerlos con el orden y el formato exactos que requiere la aplicación, diferenciando claramente entre los datos almacenados y su "proyección" o vista final.

Un error común es asumir que las filas saldrán en el mismo orden en que las guardamos con `INSERT`. Si no especifico explícitamente la cláusula `ORDER BY`, el sistema gestor de base de datos es libre de devolver los resultados en cualquier orden arbitrario debido a sus optimizaciones internas de almacenamiento.
