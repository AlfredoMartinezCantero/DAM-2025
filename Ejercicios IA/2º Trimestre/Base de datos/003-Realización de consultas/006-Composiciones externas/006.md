# Introducción:
En este ejercicio he creado una aplicación web conectándola a una base de datos MySQL existente, para gestionar la comunicación con los datos he usado a libreria `mysql.connector` encapsulando la lógica de conexión detntro de la función `obtener_conexion` para mantener el código modular.

La estructura del programa se basa en instanciar la clase `Flask` y definir rutas mediante decoradores, en la función asociada a `@app.route("/")` se inicia una conexión y creo un cursor el cual se encarga de ejecutar la sentencia SQL `SELECT * FROM matriculas_join;`.

Luego he usado el método `fetchall()` del cursor para recuperar todas las filas resultates de la consulta de una sola vez para almacenarlo en una variable.

Por último he utilizado la función `render_template` para enviar los datos recuperados a una plantilla `index.html`.

---
```
import mysql.connector
from flask import Flask, render_template

def obtener_conexion():
    return mysql.connector.connect(
        host="localhost",
        user="composiciones",  
        password="Composiciones123$",
        database="composiciones"
    )

app = Flask(__name__)

#########################################################

@app.route("/")
def inicio():
    conexion = obtener_conexion()
    
    cursor = conexion.cursor()
    
    cursor.execute("SELECT * FROM matriculas_join;")
    filas = cursor.fetchall()
    
    cursor.close()
    conexion.close()
    
    return render_template("index.html", matriculas=filas)   

if __name__ == "__main__":
    app.run(debug=True)
```

# Conlcusión:
Con este código aprendemos las bases para desarrollar backends en Python, es muy importante abrir y cerrar correctamente tanto el cursor como la conexión a la baase de datos para liberar recursos del servidor.

Un error común es dejar las credenciales de usuario y contraseña escritas dentro de la función, esto esta hecho así en este ejercicio por que es una práctica, pero en una situación real esto no debería verse de esta manera.
