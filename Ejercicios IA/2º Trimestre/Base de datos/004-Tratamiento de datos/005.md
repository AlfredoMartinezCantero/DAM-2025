# Introducción:
En este ejercicio he aprendido a manipular cadenas de texto en PHP para entender los fundamentos de la codificación y decodificación de información, he comenzado utilizando las funciones `base64_encode()` y `base64_decode()`, comprendiendo que este método convierte datos binarios en una representación ASCII segura para su transmisión, aunque no los encripta realmente.

Para profundizar en la lógica de programación he creado mis propias funciones personalizadas, `codifica9veces` y `decodifica9veces`, en ellas he implementado bucles `for` que iteran nueve veces sobre la cadena, aplicando la codificación Base64 de forma recursiva, lo que me ha permitido ver cómo la cadena resultante crece y se vuelve irreconocible al ojo humano con cada pasada, por último he bajado al nivel más básico trabajando con los valores ASCII de los caracteres, utilizando la función `ord()` para transformar cada letra de la contraseña en su valor numérico y `chr()` para realizar el proceso inverso, recuperando el carácter original a partir del número.

---
```
<?php
    // Defino una contraseña de prueba
    $contrasena = "BarBara_2025$";

    echo "<h3>1. Codificación Base64 Simple</h3>";
    
    // Codifico la cadena una sola vez
    $codificado = base64_encode($contrasena);
    echo "Original: $contrasena <br>";
    echo "Codificado: $codificado <br>";
    
    // Decodifico para comprobar que es reversible
    $decodificado = base64_decode($codificado);
    echo "Decodificado: $decodificado <br>";

    echo "<hr>";

    echo "<h3>2. Función Personalizada (Bucle 9 veces)</h3>";

    // Función que recibe una cadena y la codifica 9 veces seguidas
    function codifica9veces($cadena){
        for ($i = 0; $i < 9; $i++){
            $cadena = base64_encode($cadena);
        }
        return $cadena;
    }

    // Función que recibe una cadena y la decodifica 9 veces seguidas
    function decodifica9veces($cadena){
        for ($i = 0; $i < 9; $i++){
            $cadena = base64_decode($cadena);
        }
        return $cadena;
    }

    // Pruebo mis funciones
    $superCodificado = codifica9veces($contrasena);
    echo "Contraseña codificada 9 veces: <br>";
    // Uso wordwrap para que no se salga de la pantalla al ser muy larga
    echo wordwrap($superCodificado, 80, "<br>", true) . "<br><br>";

    $superDecodificado = decodifica9veces($superCodificado);
    echo "Contraseña recuperada: $superDecodificado <br>";

    echo "<hr>";

    echo "<h3>3. Manipulación ASCII (ord y chr)</h3>";
    
    // Obtengo el primer carácter de la contraseña
    $primerCaracter = $contrasena[0]; // Es la 'B'
    
    // Convierto el carácter a su valor numérico ASCII
    $valorAscii = ord($primerCaracter);
    echo "El valor ASCII de '$primerCaracter' es: $valorAscii <br>";
    
    // Recupero el carácter desde el número
    $caracterRecuperado = chr($valorAscii);
    echo "Si convierto $valorAscii de nuevo a carácter obtengo: $caracterRecuperado <br>";
?>
```
---
# Conclusión:
Con este código se consigue entender la base de la manipulación de strings en el lado del servidor, asi entendiendo que la codificación, como Base64 es un proceso reversible diseñado para el transporte de datos, no para la seguridad, además la implementación de las funciones de bucle me ha servido para reforzar cómo encapsular lógica repetitiva y mantener el código principal limpio y legible.

Un error en esta practica es confundir "codificar" con "encriptar", aunque al codificar 9 veces la contraseña parece segura porque es ilegible, cualquier persona que sepa que he usado Base64 puede revertir el proceso instantáneamente, por tanto he aprendido que para guardar contraseñas reales en una base de datos nunca debo usar estos métodos reversibles, sino algoritmos de hashing de un solo sentido como `password_hash()`.
