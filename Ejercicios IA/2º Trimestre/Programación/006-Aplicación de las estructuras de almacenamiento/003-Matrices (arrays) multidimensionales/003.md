#  Introducción:
En este ejercicio aprendemos a utilizar la librería `pickle` de Python para lograr la persistencia de datos, es decir, guardar información de manera permanente para que no se pierda al cerrar el programa. El objetivo es trabajar con archivos binarios y entender el concepto de serialización, que nos permite convertir estructuras de datos complejas (como listas o diccionarios) en una secuencia de bytes que se puede almacenar en el disco duro.

Para ello, utilizamos métodos clave como `pickle.dump()` para volcar los datos al archivo y `pickle.load()` para recuperarlos posteriormente. En pocas palabras, el ejercicio nos ayuda a comprender cómo los programas pueden "recordar" su estado entre diferentes ejecuciones, simulando el funcionamiento básico de una base de datos sin necesidad de SQL.

---
```
import pickle

#Creamos una lista de discos
discografia = [
    ["Robe", "Mayéutica", "Interludio"],
    ["AC/DC", "The Razors Edge", "Thunderstruck"]
]

#Recorremos la lista de discos
for disco in discografia:
    print(f"Artista: {disco[0]}, Álbum: {disco[1]}, Canción Favorita: {disco[2]}")

#Almacenamos la lista en un archivo binario
with open("discografia.bin", "wb") as archivo:
    pickle.dump(discografia, archivo)

#Recuperamos la lista desde el archivo binario
with open("discografia.bin", "rb") as archivo:
    discografia_recuperada = pickle.load(archivo)

for disco in discografia_recuperada:
    print(f"Artista: {disco[0]}, Álbum: {disco[1]}, Canción Favorita: {disco[2]}")
```
---
# Conclusión:
A través de este ejercicio llegamos a entender la diferencia entre la memoria volátil y el almacenamiento persistente, gracias al uso de pickle hemos visto lo sencillo que resulta guardar y recuperar estructuras completas sin tener que procesar texto manualmente línea por línea.
Este conocimiento sirve como base para desarrollar aplicaciones más robustas que necesiten guardar configuraciones, partidas de juegos o agendas. 

Un error común al trabajar con esta librería es olvidar abrir los archivos en modo binario `wb` para escribir o `rb` para leer, lo cual provocaría un error de ejecución ya que pickle no trabaja con texto plano, sino con flujos de bytes.
