# Introducción:
En este ejercicio ayuda para aprender a crear un script en Python diseñado para automatizar la lectura y el procesamiento de datos almacenados en archivos de texto plano, simulando la extracción de información de una base de datos exportada a formato CSV.

Para lograr esto, primero se inicia el proceso estableciendo un flujo de entrada con el archivo `clientes.csv` mediante la función `open()` en modo lectura `("r")`. 
En lugar de leer línea a línea en tiempo real, he optado por volcar el contenido completo en una lista utilizando el método `readlines()` . Posteriormente, itero sobre esta estructura con un bucle `for`, una vez dentro del ciclo, el uso del método `.strip()`, ya que permite sanear la cadena de texto eliminando el carácter de salto de línea `\n` invisible que persiste al final de cada fila.

Finalmente, el método `.split(",")` sirve para fragmentar la cadena limpia basándome en las comas, convirtiendo así una simple línea de texto en una lista de datos estructurados (como nombre, apellido, etc.) listos para ser manipulados.

---
```
#Abrimos el archivo
archivo = open("clientes.csv", "r")

#Leemos el archivo
lineas = archivo.readlines()

#Procesamos las lineas
for linea in lineas:
    #.strip() elimina el enter (\n) del final
    linea_limpia = linea.strip() 
    partido = linea_limpia.split(",")
    print(partido)

archivo.close()
```
---

# Conclusión:
Con este ejercicio he consolidado la base de la persistencia de datos en Python, aprendiendo a transformar información de texto crudo en estructuras de datos, es decir, listas que mi programa puede entender y procesar.

Un error frecuente en este tipo de operaciones es olvidar cerrar el flujo de datos con `archivo.close()`. 
Si el programa se detiene inesperadamente antes de esa línea o si simplemente se omite, el archivo puede quedar "secuestrado" por asi decirlo, en la memoria RAM, impidiendo que otros procesos puedan acceder a él o modificarlo hasta que se reinicie el entorno.
