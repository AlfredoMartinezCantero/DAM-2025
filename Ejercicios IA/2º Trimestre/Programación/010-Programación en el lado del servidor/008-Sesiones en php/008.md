# Introducción:
Este ejercicio ayuda a comprender la persistecia de información entre distintas páginas web, algo que las variabes normales no pueden hacer porque se destruyen al finalizar un script. Para ello, he usado la función `session_start()`, la cual avisa al servidor para que este reserve un espacio de memoria para el usuario o para recuperar el que ya tenía asignado.

Gracias al array `$_SESSION`, en lugar de pasar datos por la URL, he almacenado el nombre del jugador directamente en dicho array en el archivo de origen y asi, al navegar al archivo de destino e iniciar sesión he comprobado mediante la función `isset()` si la variable existía antes de mostrarla.

# Origen
---
```
<?php
    // Inicio una sesión
    session_start();

    // Declaro la variable localmente
    $nombre = "Jose Vicente";

    // Guardo el valor de la sesión
    $_SESSION['jugador'] = $nombre;
?>
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Origen - Shooter</title>
</head>
<body>
    <h1>Configuración de Jugador</h1>
    <p>El jugador <?php echo $nombre; ?> ha sido registrado en la sesión.</p>
    
    <a href="004-destino.php">Ir a ver el perfil del jugador</a>
</body>
</html>
```
---

# Destino
---
```
<?php
    // Inicio la sesión otra vez
    session_start();
?>
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Destino - Perfil</title>
</head>
<body>
    <h1>Perfil del Jugador</h1>
    <p>
        El nombre recuperado de la sesión es: 
        <strong>
            <?php 
                // Verifico si existe la variable para evitar errores
                if(isset($_SESSION['jugador'])){
                    echo $_SESSION['jugador']; 
                } else {
                    echo "No se ha identificado ningún jugador.";
                }
            ?>
        </strong>
    </p>
    
    <p><em>¡Listo para jugar!</em></p>
</body>
</html>
```
---
# Conclusión: 
Con este ejercicio se aprende a cómo mantener el estado de una aplicación web, entendiendo que el HTTP no "recuerda nada" y que gracias a las sesiones de PHP se pueden crear experiencias continuas para el usuario, permitiendo de esta manera que la web recuerde su nombre, preferencias o autenticación.

Un error común que he experimentado ha sido escribir contenido dentro del HTML antes de llamar `session_start()`. Al hacer esto PHP falla y muestra un error debido a que las sesiones necesitan enviar cookies a las cabeceras HTTP antes de que el cuerpo de la página comience a cargarse, por eso ahora sé que `session_start()` siempre debe ser la primera línea de este código.
