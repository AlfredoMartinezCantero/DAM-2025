# Introducción:
Estos ejercicios enseñan a gestionar el intercambio de información entre el navegador y el servidor mediante PHP. El primer ejercicio enseña el uso del método `GET`, comprobando que la información viaja visible a la URL.

En el segundo ejercicio se ha desarrollado hacia el método `POST`, este enseña que los datos viajan ocultos en la petición HTTP, permitiendo envíos más grandes y seguros.

Por último en el ejercicio 3 se implementa un formulario de autoprocesado. Al definir el atributo `action="?"` o dejarlo vacío, se logra que el formulario envíe los datos al mismo script que lo generó, centralizandolo en un único archivo.

---
```
<!-- EJERCICIO 1 -->
<?php
  echo $_GET['nombre'];
?>

<!-- EJERCICIO 2 -->
<?php
  echo $_POST['nombre'];
?>
 
<form action="006-post.php" method="POST">
  <p>Introduce tu nombre</p>    
  <input type="text" name="nombre">
  <input type="submit">
</form>


<!-- EJERCICIO 3 -->
<?php
  echo $_POST['nombre'];
?>
<form action="?" method="POST">
  <p>Introduce tu nombre</p>
  <input type="text" name="nombre">
  <input type="submit">
</form>
```
---
# Conclusión:
Este código ayuda a entender que `$_GET` es un array asociativo que recoge parámetros de la URL, mientras que `$_POST` recoge los datos del cuerpo de la petición. Además, al usar formularios que se envían a sí mismos, se consigue encapsular la lógica, lo que facilita el mantenimiento del código en módulos pequeños.

Un error común al trabajar con formularios es intentar imprimir `$_POST['nombre']` directamente sin comprobar si existe. Esto genera un error: `Undefined array key` la primera vez que se carga la página, esto se debe a que el usuario aún no ha pulsado "Enviar".
