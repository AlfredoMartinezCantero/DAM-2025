# Introducción:
En este ejercicio se consigue aprender a cómo modularizar la arquitectura de una página web utilizando `include`. En lugar de escribir repetitivamente la estructura HTML en cada página, lo que se consigue es separar cada elemento en fragmentos reutilizables alojados en diferentes directorios. Así conseguimos que el archivo principal se convierta en un contenedor dinámico que "llama" a las diferentes paartes que necesite para formarse.

---
```
<?php include "bloques/cabecera.php"; ?>
<p> Me encanta escuchar música, especialmente del género extremoduro, y disfruto jugando videojuegos, especialmente shooters como Halo y Souls</p>
<?php include "bloques/pie.php";  ?>
```
---
# Conclusión:
Con este ejercicio nos damos cuenta que la mayor ventaaja de usar `include` no es solo escribir menos código, sino facilitar los cambios futuros que se hagan.

Un error que he cometido es que, por ejemplo en la ruta cometí un error tipográfico, "bloques/cabeceraa.php" (puse doble "a"). Esto provocó que el servidor PHP mostrase un error indicando que no encontraba el archivo, rompiendo por completo la ejecución de la página.
