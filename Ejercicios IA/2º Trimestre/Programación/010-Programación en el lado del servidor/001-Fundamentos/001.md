# Introducción:
En este ejercicio he desarrollado un script en PHP diseñado para automatizar la toma de decisiones lógica basada en valores numéricos, para comenzar, he inicializando una variable global llamada `$edad` con un valor entero. Para evitar la repetición, he definido una función personalizada llamada `clasificarEdad`, la cual define un parámetro de entrada `$edadRecibida`.

Dentro de la función, he implementado una estructura de control `if/else` que evalúa la condición de si el argumento recibido es menor estricto que 30, dependiendo de la evaluación de verdad de esta comparación, asigno una cadena de texto diferente a una variable interna. Un punto clave de mi implementación es el uso de la sentencia `return;` en lugar de imprimir el resultado directamente dentro de la función, devuelvo el valor al flujo principal del programa. Finalmente, capturo ese retorno en la variable `$mensajeFinal` y utilizo la instrucción `echo` concatenando el valor original con el resultado procesado para mostrar la salida al usuario.

---
```
<?php
// Defino la variable con un número entero
$edad = 24;

function clasificarEdad($edadRecibida){
    // Verifico que el parámetro recibito sea menor que 30
    if ($edadRecibida < 30){
        $resultado = "Eres un jovenzuelo";
    }else{
        $resultado = "Ya no eres tan joven";
    }
    // Devuelvo el resultado
    return $resultado;
}

$mensajeFinal = clasificarEdad($edad);

// Muestro el resultado por pantalla
echo "Tengo $edad años" . $mensajeFinal;
?>
```
---
# Conclusión:
Con este código podemos aprender la base de la programación estructurada en PHP, comprendiendo la importancia de encapsular la lógica dentro de funciones, también nos enseña a separar el cálculo (la lógica dentro de la función) de la visualización (el echo final), cosa que permite crear código más limpio, ordenado y reutilizable.

Un error común es confundir el uso de `return` con `echo` dentro de una función, es decir, si hubiese puesto un `echo` dentro de `clasificarEdad`, la función perdería su versatilidad, ya que me obligaría a imprimir el texto siempre, impidiéndome guardar ese resultado en una variable o usarlo para otros cálculos.
