# Introducción:
Este ejercicio muestra cómo crear un servidor web básico utilizando Flask en Python para servir contenido HTML dinámico.

Para ello usamos Flask, un microframework web que facilita la creación de aplicaciones web en Python y permite manejar rutas y solicitudes HTTP de manera sencilla.

---
```
#1.- Instalamos flask.
# pip install flask  (windows)
# pip3 install flask --break-system-packages  (linux/macos)

#2.- Ponemos en marcha el servidor.
from flask import Flask

aplicacion = Flask(__name__)

@aplicacion.route("/")
def raiz():
  return "Esto es HTML desde Flask"
  
if __name__ == "__main__":
  aplicacion.run(debug=True)

#3.- Podemos crear HTML más complejo.
from flask import Flask

aplicacion = Flask(__name__)

@aplicacion.route("/")
def raiz():
  return '''
    <!doctype html>
    <html>
      <head>
        <title></title>
        <style>
          h1{color:red;}
        </style>
      </head>
      <body>
        <h1>Esto es HTML a tope de power</h1>
      </body>
    </html>
  '''
  
if __name__ == "__main__":
  aplicacion.run(debug=True)

#4.- HTML dinámico.
from flask import Flask

aplicacion = Flask(__name__)

@aplicacion.route("/")
def raiz():
  cadena = '''
    <!doctype html>
    <html>
      <head>
        <title></title>
        <style>
          h1{color:red;}
        </style>
      </head>
      <body>
        <h1>Esto es HTML a tope de power</h1>
  '''
       
  for dia in range(1,31):
    cadena += '<div>'+str(dia)+'</div>' 
       
  cadena += '''
      </body>
    </html>
  '''
  return cadena
 
if __name__ == "__main__":
  aplicacion.run(debug=True)
```

# Conclusión:
Este ejercicio demuestra cómo utilizar Flask para crear un servidor web que sirve contenido HTML dinámico. Hemos visto cómo instalar Flask, configurar rutas básicas y generar HTML dinámico utilizando bucles en Python. Flask es una herramienta poderosa para desarrollar aplicaciones web de manera rápida y eficiente.

Un error común al escribir código HTML dentro de cadenas en Python es olvidar cerrar las etiquetas correctamente o no manejar adecuadamente las comillas dentro del HTML. Es importante asegurarse de que la estructura del HTML sea válida para evitar problemas al renderizar la página en el navegador.
