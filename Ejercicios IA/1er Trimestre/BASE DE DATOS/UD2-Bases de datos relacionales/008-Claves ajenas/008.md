# Introducción:
En este ejercicio, enseña a utilizar claves ajenas (foreign keys) en SQL para establecer relaciones entre tablas. Creando dos tablas: "personas" y "emails". La tabla "emails" tendrá una clave ajena que hará referencia a la tabla "personas". Luego, se insertan datos en ambas tablas y se realizan una consulta que combine información de ambas tablas.

`SELECT p.Nombre, p.Apellidos, e.email` se utiliza para obtener los nombres y apellidos de las personas junto con sus correos electrónicos.

`FROM personas p` se refiere a la tabla "personas" con el alias "p".

`JOIN emails e` indica que se está realizando una unión con la tabla "emails" con el alias "e".

`ON p.Identificador = e.persona` especifica la condición de unión, donde la columna "Identificador" de la tabla "personas" debe coincidir con la columna "persona" de la tabla "emails".

---
**-- Crear las tablas "personas" y "emails".**
```
CREATE TABLE personas(
    Identificador INT PRIMARY KEY,
    Nombre VARCHAR(50),
    Apellidos VARCHAR(50)
);

CREATE TABLE emails(
    ID INT PRIMARY KEY,
    persona INT,
    email VARCHAR(100),
    FOREIGN KEY (persona) REFERENCES personas(Identificador)
);
```
**-- Insertar un registro en la tabla "personas".**
```
INSERT INTO personas (Identificador, Nombre, Apellidos)
VALUES (1, 'Alfredo', 'Martínez Cantero');
```
**-- Insertar varios registros en la tabla "emails".**
```
INSERT INTO emails (ID, persona, email)
VALUES 
(1, 1, 'alfredo@gmail.com'), 
(2, 1, 'alfredomartinezcantero@gmail.com');
```
**-- Realizar una petición cruzada para obtener los correos electrónicos junto con el nombre y apellidos.**
```
SELECT 
    p.Nombre,
    p.Apellidos,
    e.email
FROM 
    personas p
JOIN 
    emails e
ON 
    p.Identificador = e.persona;
```

# Conclusión:
Este ejercicio demuestra cómo utilizar claves ajenas para establecer relaciones entre tablas en una base de datos SQL. Al crear las tablas "personas" y "emails", se define una relación mediante la clave ajena en la tabla "emails" que referencia la tabla "personas". La inserción de datos en ambas tablas y la consulta que combina información de ambas tablas ilustran cómo se pueden recuperar datos relacionados de manera eficiente.
